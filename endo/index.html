Pipeline explained

Step 1 — Data Collection

PubMed Central Scrape

All 10,000 articles were retrieved from PubMed Central (PMC), the NIH's free full-text archive of biomedical literature. The NCBI E-utilities API was used to download complete article text — abstracts, results, discussion, and conclusions — filtered to endometriosis and related reproductive health topics.

scrape_pmc.py · download_articles.py
Step 2 — Named Entity Recognition

What is NER?

Named Entity Recognition (NER) is the process of automatically identifying and labeling specific types of things mentioned in text — genes, proteins, drugs, diseases, cell types — so a computer can reason about them. SciSpacy, a biomedical NLP library built on spaCy, was used here because it's pre-trained on scientific literature and understands terminology like "IL-6", "GPX4", and "endometrial stromal cells" that general-purpose models miss.

SciSpacy · en_core_sci_lg
Step 3 — Relation Extraction

Subject → Predicate → Object Triples

After identifying entities, the next step is understanding how they relate to each other. Claude Haiku was used via the Anthropic Batch API to read each article and extract structured triples — for example: estrogen → upregulates → aromatase or dienogest → treats → endometriosis. Each triple includes the supporting evidence sentence, confidence level, and the section of the paper it came from. This produced 459,000 unique relationships across all 10,000 articles.

extract_relations.py · claude-haiku-4-5
Step 4 — Knowledge Graph

Neo4j Graph Database

All extracted triples were loaded into Neo4j, a graph database optimized for relationship queries. Entities become nodes (270,000 unique), and relationships become edges weighted by how many articles support them. This enables queries like "what inhibits IL-6?", "what pathways does ferroptosis regulate?", or "which drugs are used for PCOS that haven't been tried for endometriosis?" — questions impossible to answer with keyword search alone.

import_to_neo4j.py · Neo4j Community
Step 5 — Clustering

Grouping Articles by Research Theme

Rather than treating all 10,000 articles as a flat list, they were grouped into 35 thematic clusters using unsupervised machine learning. Each article's extracted triples were converted into a TF-IDF vector — a numerical fingerprint of which entities and relationships it mentions most. TruncatedSVD (Latent Semantic Analysis) then reduced these to 100 dimensions to capture latent themes, and K-means clustering grouped similar articles together. Claude Haiku labeled each cluster based on its top entities.

cluster_articles.py · scikit-learn
Step 6 — Hierarchical Summarization

Three Levels of Summary

Each cluster was summarized in three passes. Level 1: Claude Haiku wrote a 4–6 sentence findings summary for every individual article from its extracted triples. Level 2: Claude Sonnet synthesized each group of 50 L1 summaries into a 200–300 word thematic rollup. Level 3: Sonnet read all L2 rollups for a cluster and wrote a structured review article — covering consensus findings, controversies, mechanistic gaps, and clinical relevance. The result is what you're reading now.

summarize_clusters.py · claude-sonnet-4-6

How to use this

📖

Read a Review

Pick a cluster from the sidebar. The Review Article section gives you a Sonnet-written synthesis of everything the literature says on that theme.

🔍

Drill Into Groups

Expand "Group Summaries" to see how findings vary across 50-article cohorts within a cluster — useful for spotting contradictions.

📄

Find Source Articles

Load individual article summaries to trace a claim back to its source paper. Each PMCID links directly to PubMed Central.

🕸️

Query the Graph

For mechanistic questions ("what upregulates GPX4?"), use the Neo4j Browser at localhost:7474 with the provided Cypher query library.
